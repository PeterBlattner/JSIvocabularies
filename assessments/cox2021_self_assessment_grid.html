Accessible — hosting and licensing 0 / 3
The vocabulary is hosted on a stable, openly accessible server — no login required, no paywall
Rule 3 — hosting must be reliable and openly available in the long term
Rule 3
A machine-readable license is declared (e.g. dcterms:license pointing to a CC-BY URI) — not just stated in plain text
Rule 3b — without a machine-readable license, automated systems treat the vocabulary as restricted
Rule 3b
Both human-readable (HTML) and machine-readable (RDF) representations are served from the same URI via content negotiation or clearly signposted download links
Rule 3c — same URI, different formats for humans and machines

Interoperable — representation and mappings 0 / 4
The vocabulary is published in W3C SKOS serialised as RDF — using skos:prefLabel, skos:definition, skos:broader/narrower as minimum
Rule 4 — SKOS is the pragmatic interoperability standard for most vocabulary types
Rule 4
Top-level concepts are mapped to an upper vocabulary (schema.org, an ISO standard, or a domain upper ontology) using skos:exactMatch or skos:broadMatch
Rule 5 — anchoring to an upper vocabulary enables cross-domain discovery
Rule 5
Explicit mapping relations to at least one related vocabulary in the same domain are present (skos:exactMatch, skos:closeMatch, skos:broadMatch)
Rule 5b — mappings are the connective tissue of the vocabulary network
Rule 5b
Preferred labels are provided with explicit language tags in at least the primary working language; multilingual labels provided where relevant
Rule 4b — plain literals without language tags are not interoperable across multilingual systems
Rule 4b

Reusable — metadata, definitions and provenance 0 / 5
Vocabulary-level metadata is complete: creator (with ORCID/ROR), publisher, version, date created, date modified, description, scope
Rule 6 — vocabulary metadata following DCTERMS and DCAT
Rule 6
Every term has a minimum of: preferred label, definition (genus-differentia preferred), and source or provenance note
Rule 7 — term-level metadata: a definition sufficient for a user to understand without expert consultation
Rule 7
Scope notes, examples, and alternate labels (synonyms) are provided where helpful for disambiguation
Rule 7b — reduces misapplication by downstream users
Rule 7b
Deprecated terms are retained with owl:deprecated and a skos:changeNote pointing to their successor — they are not deleted
Rule 8 — deletion breaks existing data that uses the old term
Rule 8
Semantic versioning is applied (Major.Minor.Patch) with a documented changelog — users can cite a specific release
Rule 8b — traceability over time
